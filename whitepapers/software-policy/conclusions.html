Conclusions

In order to maximize the benefit obtained for taxpayers money, any software contract should clarify the following on top of the usual project specifications:
  1. choice of license: preferably GPL
  2. choice of architecture: modular portable design
  3. choice of platform: preferably portable, no unnecessary platform restrictions
  4. choice of language: preferably a high level, open standard language (Examples: Python, Eiffel, C++, (Java), PHP) whenever hardware and performance allow it
  5. choice of tools: preferably no need for proprietary tools (use of proprietary tools for convenience is allowable as long as these tools are not indispensible for building, maintaining or extending this software )
  6. choice of interface: well formulated API, and preferably standard cross platform interfaces and protocols like XML-RPC, SOAP or CORBA
  7. choice of standards: whenever any aspect of the software can be implemented through or make use of an approved standard, this standard has to be used
  8. choice of 3rd party software / toolkits: Existing software modules that comply with all of the above should be reused, extended or modified in preference to re-developing. Software products not compliant with the stated specifications should no be used at all.
